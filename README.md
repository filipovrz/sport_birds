# Best Sport Byrds

Професионална уеб платформа за управление на **спортни птици** — състезателни гълъби, други гълъби и др.

## Функционалности

| Модул | Описание |
|--------|----------|
| **Регистрация** | Собственици, състезатели, развъдници; **потвърждение по имейл**; приемане на общи условия и потвърждение за 16+ години |
| **Птици** | Пръстен, вид, пол, линия, статус, родители |
| **Гълъбарници** | Множество локации, GPS на картата |
| **GPS устройства** | Регистрация, API проследяване на птици |
| **Обяви състезания** | Публикуване и записване |
| **Карта** | Гълъбарници, GPS, състезания (OpenStreetMap) |
| **Родословно дърво** | До 5 поколения, оценка на инбридинг |
| **Развъждане** | Двойки, сезони, яйца и излюпване |
| **Здраве** | Ваксини, лечения, прегледи, напомняния |
| **Тренировки** | Дистанция, метео, участващи птици |
| **Състезания** | Личен дневник (мои състезания) + публични обяви и събития |
| **Общност** | Публични профили, птици, гълъбарници (по избор на потребителя) |
| **Абонаменти** | Безплатен / Стандарт / Pro; одобрение от админ |
| **Админ панел** | Потребители, планове, плащания, настройки |
| **Супер админ** | Администратори, системни настройки |

## Изисквания

- PHP 8.2+
- MySQL 8.0+
- Apache с `mod_rewrite` (или Nginx + PHP-FPM)
- За локални тестове: [Docker Desktop](https://www.docker.com/)

## Локално тестване (Docker)

```bash
cd sport_birds
docker compose up -d --build
```

1. Начална страница: **http://localhost:8080/** (продукционен изглед). Карта на всички страници за тест: **http://localhost:8080/dev** (само при `APP_ENV=local` или `APP_DEBUG=true`).
2. Отворете **http://localhost:8080/install**
3. База данни: хост `db`, потребител `sport_birds`, парола `secret`, база `sport_birds`
4. Създайте супер админ (имейл и парола)
5. Влезте на **http://localhost:8080/login**

### Регистрация и имейл

- Новите потребители трябва да потвърдят имейла си преди вход (линк от имейла).
- Формата изисква отметка за [общи условия](/legal/terms) и потвърждение, че сте навършили **16 години**.
- Повторно изпращане: **http://localhost:8080/verify-email/pending**
- На локален Docker `mail()` често не работи — при `APP_DEBUG=true` линкът за потвърждение се показва на екрана след регистрация.

**Бърза инсталация (PowerShell):**

```powershell
powershell -File scripts/local-install.ps1
# Вход: admin@test.local / TestPass123!
```

Спиране: `docker compose down`

## Инсталация на споделен хостинг

1. Качете целия проект (или само съдържанието извън `docker/` ако няма нужда).
2. Насочете **document root** към папката `public/`.
3. Създайте MySQL база и потребител в cPanel.
4. Копирайте `.env.example` → `.env` и попълнете DB данните и `APP_URL`.
5. Отворете `https://вашият-домейн/install` и завършете инсталацията.
6. След инсталация: **забранете достъп до `/install`** (изтрийте route или защитете с парола).

### .htaccess на shared hosting

Ако приложението е в подпапка, редактирайте `public/.htaccess`:

```apache
RewriteBase /подпапка/
```

## Роли

- **user** — стандартен достъп според плана
- **admin** — управление на потребители, абонаменти, планове
- **super_admin** — пълен достъп + създаване на админи

## Планове (по подразбиране)

- **Безплатен** — до 5 птици
- **Ограничен** — до 20 птици, 20 €/месец
- **Стандарт** — до 50 птици, 40 €/месец
- **Най използван** — до 100 птици, 70 €/месец
- **Професионален** — неограничени птици, родословие, експорт, 100 €/месец

Плащанията се заявяват от потребителя и се **одобряват ръчно** от админ (банков превод / Revolut).
